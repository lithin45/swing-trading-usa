# Validation artifacts — how this strategy earns (or loses) trust

This directory is the **persistent, versioned record** the mandate (§7) and the
2026-06-10 audit (P1 #3/#5) require. Nothing in here lives only in commit
messages or session notes anymore.

## Contents

- **`trials.jsonl`** — the trial ledger: one row for every configuration
  evaluation a human looked at, append-only, duplicate-id-guarded.
  `purpose=selection` rows are what the Deflated Sharpe's N counts;
  `validation` rows are out-of-sample confirmations; `robustness` rows are
  sweep cells (looked at, never used to re-tune).
- **`sweep-<start>-<end>.md`** — ±10% parameter-sensitivity sweeps
  (generated by `scripts/sweep_sensitivity.py`): the plateau check + PSR/DSR +
  CSCV PBO for the deployed config on that window.

## The go/no-go checklist (real capital)

All of the following, simultaneously, before any non-paper deployment:

1. **Expectancy > 0 and PF ≥ 1.2 on ≥ 2 untouched OOS windows** (currently:
   2025-01→2026-06 holdout and 2017–2019 — both pass, ~+0.11R, PF ~1.27).
2. **DSR ≥ 0.95 vs the ledger's selection count** — the edge must survive its
   own selection process (see the latest sweep file for the current number and
   its var(SR)-proxy caveat).
3. **CSCV PBO < 0.5** on the relevant variant family.
4. **±10% sweep verdict: plateau** — no single threshold's ±10% move erases
   the edge.
5. **Cadence proof** — backtest entry histogram respects the ≤7/month ceiling
   with the bar (not the cap) doing most of the work in normal months.
6. **3–6 months of live paper tracking within tolerance of backtest** (fill
   quality, slippage vs the 10 bps model, realized R distribution, cadence).
7. **Loss-halt replay ON** in every number used for the decision (live gates
   refuse entries the unhalted backtest would take).

Expected live degradation per the research brief: 20–50% of backtest edge
(McLean-Pontiff). A go decision should still clear zero after that haircut.

## Rules of the road

- Every new backtest evaluation that a human looks at gets a ledger row —
  *before* deciding anything based on it.
- Sweeps are robustness checks. Re-tuning a threshold because a sweep cell
  looked better is a NEW selection trial and must be logged as one (and it
  restarts the holdout clock for that parameter).
- Negative results are recorded with the same care as positive ones.
